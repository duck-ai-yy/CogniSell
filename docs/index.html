The Problem

Long-cycle sales run on scattered memory

Industrial B2B deals take 6–18 months. During that time, relationship context fragments across tools and minds.

📧

Context Fragmentation

Key commitments buried in email thread #47. Promises made at trade shows lost in CRM notes nobody reads.

💤

Silent Decay

Contacts go cold after 90 days — silently, with no alert. By the time you notice, the relationship has cooled.

🔍

Black-Box AI

AI CRMs stuff everything into vector memory. You can't audit what the system "knows" or how confident it is.

🔄

Missed Signals

A champion changes jobs. A tender gets announced. The signals arrive — but nobody's watching.

The Solution

A knowledge graph with cognitive state

Every relationship fact is a triple with provenance, confidence, and an auditable lifecycle.

Proposed
Confirmed

1

Scan & Propose

Scan a business card at a trade show. OCR + GLiNER2 extract entities and relationships as proposed edges with confidence scores.

2

Human-in-the-Loop Sync

High-confidence facts auto-confirm silently. Low-confidence items surface in the Decision Inbox — one click to confirm, correct, or dismiss.

3

Multi-Agent Strategy

Three-role debate (Champion / Skeptic / Closer) produces approach strategies. You pick one — preferences are remembered for next time.

4

Outreach & Digest

Draft and send contextual emails. When replies arrive, the digest agent extracts new commitments and next-steps — the graph grows automatically.

5

Relationship Intelligence

90-day decay sweep highlights cold contacts. Job changes trigger instant alerts. Catch-up suggestions are grounded in graph context, not generic templates.

Architecture

Contract-first, fully decoupled

All modules communicate through the Graph Core contract. Every external I/O has a mock implementation for deterministic offline demos.

Legibility

Auditable by design

Every edge carries source, extractor, confidence, and status. No vector embeddings — every fact is human-readable and traceable.

One Schema

Three scenarios, zero silos

Card scanning, email outreach, and relationship decay all share the same triple + cognitive-state model. Decay is a time filter, not a separate schema.

Decision Inbox

Never block, always sync

Agents run continuously. Uncertain facts go to the inbox; confident facts auto-confirm. Thresholds relax as users clear cards — the system learns what matters.

Dual Implementation

Offline-first demo

Every external service (OCR, email, company data, social) has a real + mock path. Demo runs fully offline with deterministic results.

Runtime Agents

Five agents, hot-pluggable skills

Each agent composes skills from a registry. Plug in a new skill at runtime and the agent gains a new capability instantly.

Tech Stack

Minimal by conviction

Backend

FastAPI + Python

Thin API layer. Python required for GLiNER2 extraction pipeline. In-memory graph with JSON persistence.

Frontend

Vanilla HTML/JS/CSS

Zero build step. Cytoscape.js (CDN) for graph visualization. Inter font. Premium minimal design language.

Agent Orchestration

Script-controlled debate

3 sub-roles, fixed rounds. LLM fills arguments — the structure is locked. No open-ended spinning.

Extraction

GLiNER2 (Pioneer)

Named entity + relation extraction for business cards and email threads. Mock fallback for offline demo.
